# Case Study: Building a Practical Data Governance & Catalogue Framework

## Situation

Many organisations have data spread across spreadsheets, CRM systems, public datasets, APIs and BI reports. Teams often know that useful data exists somewhere, but they do not always know who owns it, what it means, whether it is trusted or which reports depend on it.

This creates problems for reporting, analytics, data products and AI-enabled use cases.

## Problem

Common issues include:

- Datasets are scattered across teams and systems
- KPI definitions are inconsistent
- Data ownership is unclear
- Reports are difficult to trust
- Data quality issues are not visible
- Teams duplicate work because existing assets are hard to find
- Organisations want to use AI, but their metadata is not ready

Without a clear catalogue and governance model, data teams spend too much time explaining, checking and manually tracing data.

## Approach

This project creates a lightweight governance and catalogue framework that helps organisations document and manage their data assets.

The approach covers:

1. Registering data sources
2. Documenting datasets
3. Defining key fields and business terms
4. Assigning data owners and stewards
5. Creating KPI definitions
6. Applying data quality rules
7. Mapping source-to-report lineage
8. Preparing metadata for future AI-enabled discovery

## Outputs

The MVP includes:

- Governance operating model
- Catalogue management process
- Data quality framework
- Dataset catalogue template
- Business glossary template
- KPI definition template
- Data quality rules template
- Sample catalogue records
- Report inventory sample
- Source-to-report lineage sample

## Example Use Case

A user wants to understand which dataset should be used for organisation-level reporting.

Using the catalogue, they can identify:

- The correct dataset
- The source system
- The dataset owner
- The steward responsible for metadata
- The fields included
- The KPI definitions linked to the dataset
- The reports using the dataset
- The data quality rules attached to the dataset
- The current status of the dataset

## Why This Matters

This type of framework creates a trust layer between raw data, reporting, analytics and AI tools.

It helps organisations move from scattered data assets to governed, searchable and reusable data products.

## Impact

A framework like this can support:

- Better trust in reports and dashboards
- Clearer ownership of data assets
- More consistent KPI definitions
- Faster onboarding for analysts and business users
- Better data quality visibility
- Stronger Microsoft Fabric and lakehouse implementation
- Improved readiness for AI assistants and self-service analytics
